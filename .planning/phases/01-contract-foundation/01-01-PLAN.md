Update Marketplace.sol to pay royalties on every buyItem settlement:

1. Add imports at top:
   ```
   import {IERC2981} from "@openzeppelin/contracts/interfaces/IERC2981.sol";
   import {IERC165} from "@openzeppelin/contracts/utils/introspection/IERC165.sol";
   ```
2. Add an internal `_deductFeesAndPay` function:
   - Takes `(address nftContract, uint256 tokenId, address payable seller, uint256 salePrice)`.
   - Use try/catch on `IERC165(nftContract).supportsInterface(type(IERC2981).interfaceId)` to check ERC-2981 support. If supported, call `IERC2981(nftContract).royaltyInfo(tokenId, salePrice)` to get `(royaltyReceiver, royaltyAmount)`.
   - Calculate commission: `(salePrice * commissionRate) / COMMISSION_DENOMINATOR`.
   - ROYL-04 guard: `require(royaltyAmount + commission <= salePrice, "Fees exceed sale price")`.
   - Pay royalty: if royaltyAmount > 0 and royaltyReceiver != address(0), transfer royaltyAmount to royaltyReceiver.
   - Calculate sellerProceeds = salePrice - commission - royaltyAmount.
   - Transfer sellerProceeds to seller. Commission stays in contract.
3. Refactor `buyItem` to use `_deductFeesAndPay` instead of inline commission math. Remove the old `commission` and `sellerProceeds` local variables. After `listing.active = false` and NFT transfer, call `_deductFeesAndPay(listing.nftContract, listing.tokenId, listing.seller, listing.price)`.
4. Add a `mapping(address => mapping(uint256 => uint256)) public activeListingByToken` for the reverse lookup needed by plan 01-05 (offer-unlist). Set it in `listItem`: `activeListingByToken[nftContract][tokenId] = listingId + 1` (offset by 1 so 0 means "no listing"). Clear it in `buyItem` and `unlistItem`: `activeListingByToken[listing.nftContract][listing.tokenId] = 0`.
5. Update the ItemSold event to include royalty: `event ItemSold(uint256 indexed listingId, address indexed buyer, uint256 price, uint256 commission, uint256 royalty)`. Emit with the royalty amount.

IMPORTANT: Keep `_removeActiveListing()` as-is. Do not change the O(N) pattern (known limitation, acceptable for MVP scale per RESEARCH.md).

- Update the "Should collect commission in marketplace" test: marketplace balance should equal commission only (royalty goes to creator, not marketplace).
- Update the "Should emit ItemSold event" test: now expects 5 args including royalty.
